# 要件定義

## プロダクト概要

**アプリ名**: Brownout  
**目的**: 日本全国10エリアの送配電事業者が公開する「でんき予報」データを iOS デバイス上で確認する  
**対象ユーザー**: 電力需給状況を日常的に確認したいユーザー（全国対応）

---

## 機能要件

### FR-01: エリア選択

- 日本全国10エリアを選択できる
- 選択エリアはアプリ再起動後も保持される（`UserDefaults` 永続化）
- 初期デフォルト: 東京

| エリア | 会社名 |
|--------|--------|
| 北海道 | 北海道電力ネットワーク |
| 東北 | 東北電力ネットワーク |
| 東京 | 東京電力パワーグリッド |
| 中部 | 中部電力パワーグリッド |
| 北陸 | 北陸電力送配電 |
| 関西 | 関西電力送配電 |
| 中国 | 中国電力ネットワーク |
| 四国 | 四国電力送配電 |
| 九州 | 九州電力送配電 |
| 沖縄 | 沖縄電力 |

### FR-02: 当日の電力需給チャート表示

- 選択エリアの当日30分間隔（九州は60分間隔）の電力データをラインチャートで表示する
- 表示する系列:
  - **最大供給力**: 当日の最大供給可能電力（青、破線）
  - **実績需要**: 実際に使われた電力（緑、実線）
  - **予測値**: 需要予測（赤、点線）
- X 軸: 0:00〜24:00 JST（3時間刻みラベル）
- Y 軸: 電力量（万kW 単位）
- チャート上に現在時刻を縦線マーカーで表示する

### FR-03: 現在の使用率表示

- 最新の実績需要 ÷ 最大供給力 × 100 を「使用率 %」として表示する
- 使用率に応じてカラーコーディング:
  - 80% 未満: 緑
  - 80〜90%: 黄
  - 90% 以上: 赤

### FR-04: データ取得・更新

- アプリ起動時に自動でデータを取得する
- エリア変更時に自動でデータを再取得する
- ツールバーの更新ボタンで手動更新できる
- プルトゥリフレッシュで手動更新できる
- データ取得中はローディングインジケーターを表示する

### FR-05: エラーハンドリング

- データ取得失敗時にエラーメッセージを表示する
- 再試行ボタンを提供する
- 各エリアの公式ページへのリンクを提供する（URL 変更への対応）

### FR-06: About 画面

- アプリ名・バージョン情報を表示する
- でんき予報まとめページ（送配電網協議会）へのリンクを提供する
- 各エリアの公式ページリンク一覧を表示する
- ツールバーの「i」ボタンからシートとして表示する

### FR-07: ローカライズ

- ベース言語: 英語
- 対応言語: 日本語
- エリア名・会社名も両言語対応

## 非機能要件

| 項目 | 要件 |
|------|------|
| 対応 OS | iOS 26 以降 |
| デバイス | iPhone（Universal 対応は任意） |
| テーマ | ライト/ダークモード両対応 |
| ネットワーク | 各社 CSV エンドポイントへの HTTPS アクセス |
| パフォーマンス | エリア切替後3秒以内にチャート表示 |
| 永続化 | 選択エリアを `@AppStorage` で保持 |
| アクセシビリティ | Dynamic Type 対応 |

---

## 廃止機能

| 機能 | 廃止理由 |
|------|---------|
| Twitter（SNS）投稿 | SNS 連携不要の方針 |
| tden.me リンク | サービス終了済み |
| Debug タブ | 開発者向け機能、不要 |
| Travis CI | 使用していない |
